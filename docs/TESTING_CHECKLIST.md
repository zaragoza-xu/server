# 测试要点清单

本文档面向本项目测试开发者，给出按优先级划分的测试要点与执行建议。

## P0 核心正确性（必须覆盖）

### 1. 协议 type 字符串契约

- 仅接受字符串命令名：`register/login/create_room/join_room/leave_room/list_rooms/send_message`
- `type` 非字符串时，返回 `ERROR`
- `type` 未知字符串时，返回 `ERROR`

### 2. Envelope 统一返回结构

每个响应必须包含以下字段：

- `code`
- `message`
- `data`

断言建议：

- 成功：`code & SUCCESS` 非 0
- 业务失败：`code & FAIL` 非 0，且 `message` 非空
- 系统错误：`code & ERROR` 非 0，细节位（如 `DESERIALIZE_FAIL`）符合预期

### 3. 注册/登录主流程

- 注册成功后可登录
- 使用不存在 uid 登录时应返回错误，不可崩溃
- 重复注册、重复登录行为应符合当前实现约束（并保持一致）

### 4. 房间生命周期

- 创建房间成功后返回有效 `roomId`
- 加入房间成功后成员列表包含当前用户
- 离开房间后成员计数减少
- 空房间自动删除

## P1 边界与异常（高价值）

### 1. 参数边界

- `uid` 为空
- `roomName` 为空
- `roomId=-1` 或不存在
- `maximumPeople` 为 `0/负数/极大值`

### 2. JSON 输入异常

- 缺失字段
- 多余字段
- 字段类型错误（如数字字段传字符串）
- 非法 JSON（语法错误）

### 3. 网络异常

- 客户端中途断连
- 半包/粘包输入
- 超长消息
- 断连后用户状态是否正确清理（含房间成员）

### 4. 容量约束

- 房间满员后拒绝加入
- 并发加入同一房间时容量限制仍生效

## P2 并发与稳定性（建议覆盖）

### 1. 并发 join/leave

- 同一用户快速 join/leave 多次
- 多用户并发 join 同一房间
- 成员计数与成员列表一致性

### 2. 并发断连与广播

- 广播过程中有用户断连
- 在线用户可继续收到消息
- 无崩溃、无明显卡死

### 3. 长稳运行

- Soak test（建议 1-4 小时）
- 关注内存增长、连接资源释放、异常日志频度

## 协议回归测试（建议自动化）

### 1. 快照回归

- 每个命令建立固定输入与固定响应结构快照
- 重点字段：`code/message/data`

### 2. 契约一致性

- 强约束 `type` 使用字符串
- 强约束业务数据在 `data` 内

## 分层测试建议

### 1. 单元测试

- `Protocol` 编解码（CommandType/Envelope/Req/Rsp）
- `Room` 成员增删和容量上限逻辑

### 2. 集成测试

- 通过 TCP 客户端进行端到端命令流测试
- 覆盖 register/login/create/join/leave/list

### 3. 压测与混沌测试

- 大量并发连接
- 随机断连
- 随机非法报文注入

## 用例执行模板（建议）

每条测试用例建议记录：

- 用例编号
- 前置条件
- 输入报文
- 预期响应
- 关键断言
- 实际结果
- 日志片段

## 里程碑建议

- 里程碑 A：P0 全通过
- 里程碑 B：P1 全通过
- 里程碑 C：P2 压测通过且无高优缺陷
